OffLink ကိုယ်ရေးအချက်အလက်မူဝါဒ

နောက်ဆုံးအပ်ဒိတ်: ၂၀၂၆ ခုနှစ် အပြီလ ၂၂ ရက်

H Labs Works ("ကျွန်ုပ်တို့") သည် မိုဘိုင်းအပ်ပလီကေးရှင်း "OffLink" ("အပ်") ကို ဖွင့်လှစ်ထားပါသည်။ ဤကိုယ်ရေးအချက်အလက်မူဝါဒသည် ကျွန်ုပ်တို့ သင်၏ သတင်းအချက်အလက်ကို မည်သို့ စုဆောင်း၊ အသုံးပြုနှင့် ကာကွယ်သည်ကို ရှင်းပြသည်။

၁။ အပ်အကြောင်း

OffLink သည် Wi-Fi hotspot နှင့် WebRTC နည်းပညာကို အသုံးပြုသော အင်တာနက်ချိတ်ဆက်မှုမလိုအပ်သည့် အဖွဲ့ဝိုင်းသံကဏ္ဍဆက်ကြောင်း အပ်ပလီကေးရှင်းဖြစ်သည်။ အသံဒေတာကို စက်ပစ္စည်းများကြားတွင် ဒေသဆိုင်ရာ ထုတ်လွှင့်ပြီး ပြင်ပဆာဗာများသို့ မပို့ပါ။

၂။ ကျွန်ုပ်တို့ စုဆောင်းသော သတင်းအချက်အလက်

၂.၁ အပ်မှ တိုက်ရိုက်စုဆောင်းသော သတင်းအချက်အလက်

သတင်းအချက်အလက်ရည်ရွယ်ချက်သိုလှမ်းမှု
အသုံးပြုသူအမည် (ရွေးချယ်နိုင်)ခေါ်ဆိုမှုအတွင်း အဖွဲ့ဝင် ခွဲခြားဖော်ထုတ်ခြင်းစက်ပစ္စည်းတွင်သာ (SharedPreferences)
အုပ်စုသတင်းအချက်အလက် (အုပ်စုအမည်၊ SSID၊ စကားဝှက်)အုပ်စုခေါ်ဆိုမှု ဖွဲ့စည်းမှုနှင့် ချိတ်ဆက်မှုစက်ပစ္စည်းတွင်သာ (SharedPreferences)
ရမှတ်လက်ကျန်နှင့် မှတ်တမ်းအပ်တွင်းရမှတ်စနစ် စီမံခန့်ခွဲမှုစက်ပစ္စည်းတွင်သာ (ကုဒ်ဝှက်ထား)
စက်ပစ္စည်းသတင်းအချက်အလက် (OS ဗာဆင်၊ ဘက်ထရီ၊ မှတ်ဉာဏ်)host ရွေးချယ်မှု အကဲဖြတ်ခြင်းခေါ်ဆိုမှု session အတွင်းသာ (မကြာမြင့်)

အသံဒေတာအကြောင်း: ခေါ်ဆိုမှုအသံကို WebRTC (DTLS-SRTP) ဖြင့် ကုဒ်ဝှက်ကာ ဒေသဆိုင်ရာကွန်ရက်ပေါ်ရှိ စက်ပစ္စည်းများကြားတွင်သာ ထုတ်လွှင့်သည်။ အသံဒေတာကို ကျွန်ုပ်တို့ ဆာဗာများ သို့မဟုတ် မည်သည့် ပြင်ပဆာဗာသို့မဆို မပို့ပါ။

၂.၂ တတိယပါတီဝန်ဆောင်မှုများမှ စုဆောင်းသော သတင်းအချက်အလက်

အပ်သည် အောက်ပါ တတိယပါတီဝန်ဆောင်မှုများကို အသုံးပြုသောကြောင့် ၎င်းတို့သည် သီးခြားစွာ သတင်းအချက်အလက်များ စုဆောင်းနိုင်သည်။

Google AdMob (ကြော်ငြာ)

  • ကြော်ငြာများ ပြသရန်အတွက် Google သည် ကြော်ငြာ ID (Android: AAID)၊ စက်ပစ္စည်းသတင်းအချက်အလက်၊ ခန့်မှန်းတည်နေရာနှင့် အပ်အသုံးပြုမှုဒေတာများ စုဆောင်းနိုင်သည်။
  • စုဆောင်းထားသောဒေတာကို ပုဂ္ဂိုလ်ရေးဆိုင်ရာ ကြော်ငြာများ ပြသရန် အသုံးပြုနိုင်သည်။
  • အသေးစိတ်: Google ကိုယ်ရေးမူဝါဒ

Firebase (အမည်မသိ အတည်ပြုခြင်း၊ Cloud Firestore၊ Cloud Messaging)

  • အပ်သည် ကိုယ်ရေးအချက်အလက်နှင့် ချိတ်ဆက်မဆက်ဘဲ ဆက်ရှင်များကို ခွဲခြားဖို့ Firebase အမည်မသိ အတည်ပြုခြင်းကို အသုံးပြုပါသည်။ အကောင့်ဖွင့်ခြင်း သို့မဟုတ် အီးမေးလ်/စကားဝှက် မလိုအပ်ပါ။
  • Push အကြောင်းကြားချက်များ ဖွင့်ရန် အုပ်စုဝင်အချက်အလက် (အုပ်စု ID၊ ဖော်ပြမည့်အမည်၊ FCM token) ကို Cloud Firestore တွင် သိုလှမ်းပါသည်။ အုပ်စုမှ ထွက်သည် သို့မဟုတ် အပ်ကို ဖျက်သိမ်းသည့်အခါ ဒေတာများ အလိုအလျောက် ဖျက်ပါမည်။
  • Firebase Cloud Messaging (FCM) ကို အပ်ကာနောက်ကွယ်တွင် ရှိနေချိန် ခေါ်ဆိုမှုဖိတ်ကြားချက် အကြောင်းကြားချက်များ ပို့ရန် အသုံးပြုပါသည်။
  • အသေးစိတ်: Firebase ကိုယ်ရေးနှင့်လုံခြုံရေး

Firebase Crashlytics (ပျက်ကျမှု အစီရင်ခံခြင်း)

  • အပ်၏ တည်ငြိမ်မှု တိုးတက်ရန် ပျက်ကျမှုအချက်အလက်၊ မသေဆုံးသောအမှားများနှင့် breadcrumb log များကို Firebase Crashlytics သို့ ပို့ပါသည်။
  • သင်ရဲ့ ကိုယ်ရေးကာကွယ်ရန် အသုံးပြုသူ ခွဲခြားသတ်မှတ်ချက်များ (စက်ပစ္စည်း ID စသည်) ကို ပို့မည်မဟုတ်ပါ။ Crashlytics သည် Firebase Installation ID သာ အသုံးပြု၍ လည်ပတ်ပါသည်။
  • ဒေတာများကို Google ၏ Firebase ဝန်ဆောင်မှုများသို့ ပို့ပြီး သိုလှမ်းပါသည်။ EU ဒေတာနေထိုင်မှုကို ပံ့ပိုးမပေးပါ; Google ၏ EU-US ဒေတာကိုယ်ရေးဘောင် (DPF) ကျင့်သုံးပါသည်။
  • အသေးစိတ်: Firebase ကိုယ်ရေးနှင့်လုံခြုံရေး

၃။ တည်နေရာသတင်းအချက်အလက်

အပ်သည် Wi-Fi hotspot လုပ်ဆောင်ချက်ကို ထိန်းချုပ်ရန်နှင့် SSID သတင်းအချက်အလက်ရယူရန် Android တည်နေရာခွင့်ပြုချက် (ACCESS_FINE_LOCATION) ကို အသုံးပြုသည်။ ဤခွင့်ပြုချက်သည် Wi-Fi ဆိုင်ရာ API များအတွက် Android OS မှ လိုအပ်ပြီး သင်၏ ပထဝီဝင်တည်နေရာကို ခြေရာခံ၊ မှတ်တမ်းတင် သို့မဟုတ် ထုတ်လွှင့်ရန် မသုံးပါ။

၄။ ကင်မရာ

အပ်သည် QR ကုဒ်စကင်ဖတ်ရန် ကင်မရာခွင့်ပြုချက်ကို အသုံးပြုသည်။ ရိုက်ကူးသောပုံများကို QR ကုဒ်ခွဲခြမ်းစိတ်ဖြာမှုအတွက်သာ အသုံးပြုပြီး သိုလှမ်းခြင်း သို့မဟုတ် ထုတ်လွှင့်ခြင်း မပြုပါ။

၅။ မိုက်ခရိုဖုန်း

အပ်သည် အဖွဲ့ဝိုင်းသံကဏ္ဍဆက်ကြောင်းများအတွက် မိုက်ခရိုဖုန်းခွင့်ပြုချက်ကို အသုံးပြုသည်။ မှတ်တမ်းတင်ထားသောအသံကို တူညီသောဒေသဆိုင်ရာကွန်ရက်ပေါ်ရှိ အဖွဲ့ဝင်များသို့သာ အချိန်နှင့်တပြေးညီ ထုတ်လွှင့်ပြီး မည်သည့် စက်ပစ္စည်း သို့မဟုတ် ဆာဗာတွင်မဆို မှတ်တမ်းတင် သို့မဟုတ် သိုလှမ်းခြင်း မပြုပါ။

၆။ တတိယပါတီများနှင့် ဒေတာမျှဝေခြင်း

အောက်ပါကိစ္စများမှလွဲ၍ သင်၏ ကိုယ်ရေးအချက်အလက်ကို တတိယပါတီများနှင့် မမျှဝေပါ:

  • အပိုင်း ၂.၂ တွင် ဖော်ပြထားသော သတင်းအချက်အလက်ကို ကြော်ငြာတင်ဆက်မှုအတွက် Google AdMob သို့ အလိုအလျောက် ပေးပို့ပါသည်။
  • ဥပဒေအရ လိုအပ်သောအခါ
  • သင်၏ သဘောတူညီချက်ဖြင့်

၇။ ကြော်ငြာ ပယ်ဖျက်ခြင်း

ပုဂ္ဂိုလ်ရေးဆိုင်ရာ ကြော်ငြာများ မလိုချင်ပါက သင်၏ ဆက်တင်များကို ပြောင်းလဲနိုင်သည်:

  • Android: ဆက်တင်များ → Google → ကြော်ငြာ → "ကြော်ငြာ ပုဂ္ဂိုလ်ရေးဆိုင်ရာ ပြင်ဆင်မှု ပိတ်ထားရန်" ဖွင့်ပါ
  • iOS: ဆက်တင်များ → ကိုယ်ရေးနှင့် လုံခြုံရေး → ခြေရာခံခြင်း → "အပ်ပလီကေးရှင်းများကို ခြေရာခံမှုတောင်းဆိုခွင့်ပေးရန်" ပိတ်ပါ

၈။ ဒေတာ သိမ်းဆည်းမှုနှင့် ဖျက်ပစ်ခြင်း

  • အပ်မှ တိုက်ရိုက်သိုလှမ်းသော ဒေတာများ အားလုံးကို သင်၏ စက်ပစ္စည်းတွင်သာ သိမ်းဆည်းထားသည်။
  • အပ်ကို ဖျက်သိမ်းခြင်းသည် ဒေသဆိုင်ရာ ဒေတာအားလုံးကို ဖျက်ပစ်မည်။
  • Google AdMob မှ စုဆောင်းသောဒေတာအတွက် Google ၏ ကိုယ်ရေးမူဝါဒကို ကြည့်ပါ။

၉။ ကလေးများ၏ ကိုယ်ရေးကာကွယ်မှု

အပ်သည် ၁၃ နှစ်အောက် ကလေးများအတွက် မဟုတ်ပါ၊ ၁၃ နှစ်အောက် ကလေးများ၏ ကိုယ်ရေးအချက်အလက်ကို တမင် မစုဆောင်းပါ။

၁၀။ ဤမူဝါဒ ပြောင်းလဲမှုများ

လိုအပ်သလို ဤကိုယ်ရေးအချက်အလက်မူဝါဒကို အပ်ဒိတ်လုပ်နိုင်သည်။ အရေးကြီးသော ပြောင်းလဲမှုများကို အပ်တွင်းအကြောင်းကြားချက် သို့မဟုတ် ဤစာမျက်နှာမှ အကြောင်းကြားပါမည်။

၁၁။ ဆက်သွယ်ရန်
